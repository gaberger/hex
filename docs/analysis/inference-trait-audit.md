# P1.1 — Inference Trait Surface Audit

**Workplan**: `wp-hex-standalone-dispatch`
**ADR**: `ADR-2604112000` (Hex Self-Sufficient Dispatch / Standalone Mode)
**Author**: planner (P1.1)
**Date**: 2026-04-11
**Consumer**: P1.2 (`hex-core/src/ports/inference.rs` consolidation)

---

## 1. Executive summary

- `hex-core::ports::inference::IInferencePort` **already exists** (`hex-core/src/ports/inference.rs:93`). It has `complete`, `stream`, and `capabilities` — roughly what the ADR asks for, minus an explicit `health()` method. The ADR requirement is therefore **refine-in-place**, not greenfield rewrite.
- **Exactly one** implementation ships today: `SpacetimeInferenceAdapter` in `hex-agent/src/adapters/secondary/spacetime_inference.rs:67`. There are **zero** `IInferencePort` impls anywhere in `hex-nexus`. The nexus binary has no inference backend at all — confirming ADR-2604112000 Context §2.
- `ClaudeCodeInferenceAdapter` **does exist** in `hex-agent/src/adapters/secondary/claude_code_inference.rs:13`, but it is a one-method bypass shim (`run_task`), **not** an `IInferencePort` implementation, and it lives in hex-agent rather than hex-nexus. P4 must either move/relocate it or rewrite it as a real port impl.
- Two adjacent traits (`IInferenceRouterPort`, `IInferenceTaskStatePort`) are **not** inference backends — they are a fleet router and a SpacetimeDB reducer port. Both must survive consolidation. Only `IInferencePort` itself is in scope for P1.2.
- No ad-hoc or half-formed inference trait exists — the table-access traits under `spacetime_bindings/` are generated SpacetimeDB client code and must not be touched.

## 2. Existing inference traits

### 2.1 `hex_core::ports::inference::IInferencePort` — **canonical, in scope**

- **File**: `hex-core/src/ports/inference.rs:93`
- **Crate**: `hex-core` (ports layer, zero deps beyond `async_trait` + `serde`)
- **Signature**:
  ```rust
  #[async_trait]
  pub trait IInferencePort: Send + Sync {
      async fn complete(&self, request: InferenceRequest)
          -> Result<InferenceResponse, InferenceError>;
      async fn stream(&self, request: InferenceRequest)
          -> Result<Box<dyn futures_stream::Stream<Item = StreamChunk> + Send + Unpin>,
                    InferenceError>;
      fn capabilities(&self) -> InferenceCapabilities;
  }
  ```
- **Associated domain types** (same file):
  - `InferenceRequest { model, system_prompt, messages, tools, ... }`
  - `InferenceResponse`, `StreamChunk`, `InferenceCapabilities { model_id, tier, context_window }`
  - `ModelTier { Opus, Sonnet, Haiku, Local }`
- **Error enum**: `InferenceError` with variants `RateLimited`, `BudgetExceeded { used, limit }`, `ProviderUnavailable`, `ApiError { status, body }`, `Network`, `UnknownProvider`.
- **Custom stream indirection**: file defines a local `pub mod futures_stream { pub trait Stream { ... } }` to avoid depending on the `futures` crate. This is a correct zero-dep choice but is awkward — see §5.
- **Claude-specific assumptions baked in**: **none** in the trait itself. `ModelTier::{Opus, Sonnet, Haiku}` is Anthropic-flavoured but doesn't force Claude as a backend.
- **No `health()` method** — the ADR explicitly calls for one; P1.2 must add it.
- **No `name()` / `backend_id()` method** — arguably useful for routing/logging but neither the ADR nor current call sites need it.

### 2.2 `hex_nexus::ports::inference_router::IInferenceRouterPort` — **OUT OF SCOPE, keep**

- **File**: `hex-nexus/src/ports/inference_router.rs:16`
- **Purpose**: fleet-level routing (select_server / route_request / has_model / fleet_capacity) — ADR-040, NOT a per-provider backend. Consumes `IRemoteRegistryPort` + `IAgentTransportPort`, returns `InferenceServer` from `crate::remote::transport`.
- **Implementation**: `hex-nexus/src/adapters/inference_router.rs:30` — `InferenceRouterAdapter` implements it.
- **Do not delete, do not rename**. It's orthogonal to `IInferencePort`: the router picks a *server*, the port talks to one *backend*. P1.2 should leave it alone.

### 2.3 `hex_nexus::ports::state::IInferenceTaskStatePort` — **OUT OF SCOPE, keep**

- **File**: `hex-nexus/src/ports/state.rs:623`
- **Purpose**: thin wrapper over `inference_task_create/claim/complete/fail` SpacetimeDB reducers. This is how Path B (ADR-2604010000) queues tasks in the `inference-gateway` WASM module for the outer Claude Code shell to consume.
- **Implementation**: `hex-nexus/src/adapters/spacetime_state.rs:976` and `:1700`.
- **Do not confuse with a backend**. It persists *requests* to SpacetimeDB; it does not *perform* inference.

### 2.4 SpacetimeDB-generated table-access traits — **OUT OF SCOPE, do not touch**

Auto-generated by `spacetime generate`, purely mechanical row-access glue:

- `hex-nexus/src/spacetime_bindings/secret_grant/inference_endpoint_table.rs:25` `InferenceEndpointTableAccess`
- `hex-nexus/src/spacetime_bindings/inference_gateway/inference_request_table.rs:25` `InferenceRequestTableAccess`
- `hex-nexus/src/spacetime_bindings/inference_gateway/inference_response_table.rs:25` `InferenceResponseTableAccess`
- `hex-nexus/src/spacetime_bindings/inference_gateway/inference_stream_chunk_table.rs:25` `InferenceStreamChunkTableAccess`
- `hex-nexus/src/spacetime_bindings/inference_gateway/inference_provider_table.rs:25` `InferenceProviderTableAccess`
- (plus `*_query_table_access` variants at line ~134/151 of each file)

**P1.2 must not edit these.** They are regenerated from `spacetime-modules/*.rs`.

## 3. Existing inference adapters

### 3.1 `SpacetimeInferenceAdapter` — **real, the only IInferencePort impl**

- **File**: `hex-agent/src/adapters/secondary/spacetime_inference.rs:67`
- **Crate**: `hex-agent` (not hex-nexus)
- **Implements**: `hex_core::ports::inference::IInferencePort`
- **Backend**: SpacetimeDB reducers in `inference-gateway` WASM module (writes an `inference_request` row, subscribes to the corresponding `inference_response` / `inference_stream_chunk` rows).
- **Status**: appears wired — this is the path hex-agent uses when it is the worker under a nexus that has SpacetimeDB running. It is not stubbed.
- **Relevance to P1.2**: confirms the trait shape is already consumable. Any shape change in P1.2 **must** preserve compatibility with this impl or include a matching edit. Scope note: P1.2 is `layer: ports` — if touching `spacetime_inference.rs` is required, that belongs in a different task.

### 3.2 `ClaudeCodeInferenceAdapter` — **exists but is NOT an IInferencePort impl**

- **File**: `hex-agent/src/adapters/secondary/claude_code_inference.rs:13`
- **Shape**:
  ```rust
  pub struct ClaudeCodeInferenceAdapter { project_dir: String }
  impl ClaudeCodeInferenceAdapter {
      pub fn new(project_dir: impl Into<String>) -> Self { ... }
      pub async fn run_task(&self, prompt: &str) -> Result<String, String> { ... }
  }
  ```
- **Backend**: spawns `claude -p <prompt> --output-format text --dangerously-skip-permissions` in `project_dir` via `tokio::process::Command`. Returns stdout as a `String`.
- **Does NOT implement `IInferencePort`**. There is no `impl IInferencePort for ClaudeCodeInferenceAdapter`.
- **Call site**: `hex-agent/src/main.rs:710` — only used on the `--prompt` one-shot bypass branch when `is_claude_code_session()` is true. Not reachable from hex-nexus.
- **Helpers in same file**: `is_claude_code_session()` (reads `CLAUDECODE` and `CLAUDE_CODE_ENTRYPOINT`), `which_claude()` (PATH lookup).
- **Relevance to P1.2**: P1.2 does NOT need to touch this file. P4 owns the rework. But P1.2 **should** define the trait in a shape that this adapter can cleanly grow into (i.e. `complete()` must accept a prompt-shaped request; `run_task` maps to a `complete()` that returns the stdout payload as a single `InferenceResponse` text block with `StopReason::EndTurn`).

### 3.3 `InferenceRouterAdapter` — real, router-level, not a backend

- **File**: `hex-nexus/src/adapters/inference_router.rs:9`
- **Implements**: `IInferenceRouterPort` (not `IInferencePort`)
- **Stub markers**: `route_request` currently returns
  ```rust
  Err(TransportError::Protocol(
      "Inference routing not yet wired to transport — requires orchestrator (T3-4)".into(),
  ))
  ```
  at `hex-nexus/src/adapters/inference_router.rs:~110`. `select_server` / `has_model` / `fleet_capacity` are real. **This stub is not P1.2's problem** — it's a routing gap, not a port gap.
- **Backend**: delegates to `IRemoteRegistryPort` for fleet discovery.

### 3.4 No `OllamaInferenceAdapter`, no `OpenAIInferenceAdapter`

`find hex-nexus -name '*ollama*' -o -name '*openai*' -o -name '*anthropic*'` returns zero inference backends under `hex-nexus/src/adapters/`. P3 is greenfield. Confirms ADR-2604112000 Context §2 verbatim.

### 3.5 No concrete `IInferencePort` impl inside hex-nexus at all

`grep -rn 'impl .* for .* ::IInferencePort' hex-nexus` → zero hits. This is the headline finding: **the nexus binary ships without any inference backend**. Every inference path currently relies on either (a) hex-agent as a subprocess that happens to own a `SpacetimeInferenceAdapter`, or (b) the Claude Code outer shell doing the inference via Path B.

## 5. Recommended consolidation for P1.2

The audit's recommendation is **refine-in-place**, not rewrite. Specifically:

### 5.1 Keep the existing trait and types

`hex-core/src/ports/inference.rs` stays the canonical home. `InferenceRequest`, `InferenceResponse`, `StreamChunk`, `InferenceCapabilities`, `ModelTier`, `InferenceError` all stay.

### 5.2 Add `health()`

ADR-2604112000 Phase 1 requires `health() -> Result<HealthStatus>`. Proposed shape:

```rust
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum HealthStatus {
    Healthy,
    Degraded { reason: String },
    Unreachable { reason: String },
}

#[async_trait]
pub trait IInferencePort: Send + Sync {
    async fn complete(&self, request: InferenceRequest)
        -> Result<InferenceResponse, InferenceError>;

    async fn stream(&self, request: InferenceRequest)
        -> Result<Box<dyn futures_stream::Stream<Item = StreamChunk> + Send + Unpin>,
                  InferenceError>;

    /// Ping the backend. Ollama: HTTP GET /. ClaudeCode: `claude --version`.
    /// Spacetime: reducer roundtrip.
    async fn health(&self) -> Result<HealthStatus, InferenceError>;

    fn capabilities(&self) -> InferenceCapabilities;
}
```

Adding a method is a breaking change for `SpacetimeInferenceAdapter`. P1.2 must include a stub impl on that adapter that returns `HealthStatus::Healthy` (or performs a real ping) so `cargo check -p hex-core -p hex-nexus` still succeeds at the phase gate. This is a **one-line touch** in `hex-agent/src/adapters/secondary/spacetime_inference.rs` — worth noting in the P1.2 task description if it is not already.

### 5.3 Add `MockInferencePort` in `hex-core/src/ports/inference/mock.rs`

Workplan explicitly calls for this. Pure in-memory mock, feature-gated behind `#[cfg(any(test, feature = "mock"))]`. Used by P2.3 (standalone composition tests) and P5 (acceptance gate). Returns fixed `InferenceResponse` for `complete`, empty stream for `stream`, `HealthStatus::Healthy` for `health`.

### 5.4 Consider simplifying the stream return type

The current `Box<dyn futures_stream::Stream<Item = StreamChunk> + Send + Unpin>` with a hand-rolled `futures_stream::Stream` trait is awkward and makes consumers re-implement poll semantics. Options for P1.2, in order of preference:

1. **Leave it alone.** Zero blast radius, ADR doesn't require the change. Recommended unless P1.2 has spare budget.
2. Switch to `futures_core::Stream` (via `futures-core = { version = "0.3", default-features = false }`) — `futures-core` is no-std and has zero transitive deps. Matches Rust ecosystem conventions. `hex-core` adds one tiny dep.
3. Replace the boxed stream with a `tokio::sync::mpsc::Receiver<StreamChunk>`. Simpler but forces tokio into `hex-core`.

**P1.2 recommendation**: option 1 (defer). The ADR says ship the minimum viable port for P3/P4; option 2 or 3 can come later behind a separate ADR if it turns out to hurt ergonomics during the Ollama implementation.

### 5.5 Do NOT add a `name()` / `backend_id()` method unless P3 needs it

The router (`IInferenceRouterPort`) handles backend identification today. Adding a `fn name(&self) -> &str` to `IInferencePort` would overlap with `InferenceCapabilities::model_id`. P1.2 should skip it.

### 5.6 Error enum additions

ADR P3 Ollama adapter needs: `Unreachable` (connection refused), `UnknownModel` (Ollama 404), `Upstream` (5xx). Today we have `ProviderUnavailable`, `UnknownProvider`, `ApiError { status, body }`. Suggested delta:

```rust
pub enum InferenceError {
    // ... existing variants ...
    #[error("Backend unreachable: {0}")]
    Unreachable(String),                     // NEW — connection refused, DNS failure, etc.
    #[error("Unknown model: {0}")]
    UnknownModel(String),                    // NEW — distinct from UnknownProvider
    #[error("Timeout after {seconds}s")]
    Timeout { seconds: u64 },                // NEW — so adapters don't smuggle it into Network(String)
}
```

`ProviderUnavailable` can stay for backend-reported 503. `UnknownProvider` can stay for router-level lookup misses. `ApiError` keeps the catch-all 4xx/5xx path. Non-breaking addition: existing match arms need `_ =>` or explicit new arms, but no call site uses exhaustive matching today (grepped — only `thiserror::Error` derives it, which is fine).

## 6. Ad-hoc redundant traits to delete

**None.** Every grep hit resolved to either (a) the canonical `IInferencePort`, (b) legitimate non-backend traits (router, task-state, generated table-access), or (c) real adapter impls. There is nothing stale to clean up. The P1.2 task description's line "Delete any ad-hoc inference traits the audit flagged as redundant" becomes a no-op: **no ad-hoc traits exist**.

If P1.2 still wants to make a symbolic deletion to shrink surface area, the only candidates are the unused `futures_stream::Stream` helper module (§5.4 option 2 would remove it) — but that is an ergonomic refinement, not a redundancy cleanup, and the recommendation here is to defer.

## 7. Surprises / gotchas for P1.2

1. **The trait already exists.** Don't greenfield it. The P1.2 task description reads like it's designing a new trait; the audit says it's adding `health()` + a mock to an existing one.
2. **The only existing impl lives in `hex-agent`, not `hex-nexus`.** Adding `health()` forces a matching edit in `hex-agent/src/adapters/secondary/spacetime_inference.rs`. That file is outside P1.2's stated `files:` list in the workplan. P1.2's executor (`hex-coder`) must either (a) amend the `files:` list when it starts, or (b) the operator must approve the amendment. If the gate is `cargo check -p hex-core -p hex-nexus` (as workplan says), `hex-agent` is technically unaffected by the check — **so the gate as written would pass with a broken hex-agent**. Recommend P1.2 runs `cargo check --workspace` locally even if the gate is narrower, or the gate is widened.
3. **`hex-nexus/src/ports/mod.rs:36` already re-exports `hex_core::ports::inference`.** No wiring changes needed in `hex-nexus` to make the trait available. P1.2's `hex-core/src/ports/mod.rs` touch is confirming existing module registration, not adding new `pub mod inference;`.
4. **`ClaudeCodeInferenceAdapter` is in hex-agent, not hex-nexus.** The workplan P4 says "Rework `ClaudeCodeInferenceAdapter` … hex-nexus/src/adapters/inference/claude_code.rs". This is a **move** across crates, not an edit in place. P1.2 doesn't need to act on this, but the P1.2 → P4 handoff should flag it.
5. **`InferenceRouterAdapter::route_request` is a real stub** (returns hardcoded `Err` at `hex-nexus/src/adapters/inference_router.rs:~110`). This is a separate gap from ADR-2604112000 — the ADR does not close it. If P3's Ollama adapter is called through the router path, the router stub will still reject the call. Recommend P3's integration tests bypass the router and construct the Ollama adapter directly.
6. **The `futures_stream::Stream` custom trait** (`hex-core/src/ports/inference.rs:~130`) means any P3/P4 adapter implementing `stream()` has to write its own pin-poll loop. This is not a bug, but it will bite the Ollama implementer. Worth warning in the P3 task notes.
7. **No `#[cfg(test)] mod tests` exists today in `hex-core/src/ports/inference.rs`.** P1.2's mock can double as its own unit test, but there is no existing test scaffolding in that file to extend.
