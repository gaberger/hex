Maintain context across the session. Store key information in memory using structured session notes.

Session notes should include:
- **Title**: short distinctive 5-10 word description
- **Current State**: what is actively being worked on
- **Task Specification**: what the user asked to build
- **Files and Functions**: important files and why they are relevant
- **Workflow**: bash commands and execution order
- **Errors & Corrections**: what failed and how it was fixed
- **Learnings**: what worked well, what not to try again
